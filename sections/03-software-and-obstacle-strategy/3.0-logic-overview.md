This section details Cheese's software architecture, covering the platform and libraries, a function-level breakdown of the code, the priority-based execution model, and the two control systems that drive the robot. The main navigation code is implemented in **Python 3 using the ev3dev2 framework, running directly on the EV3 brick.** A second processor, an Arduino Nano, handles the HuskyLens camera in C++, though vision is not yet integrated into the navigation loop.

Every section after this overview focuses on a single subsystem in full technical detail.

Cheese uses the following ev3dev2 modules and standard Python libraries:

| Library / Module | Purpose |
| :--- | :--- |
| `ev3dev2.motor` | Controls the drive and steering motors (`LargeMotor`, `MediumMotor`), including speed and position commands. |

## вқҖ 3.0.3 Design Principle: Minimum Sufficient Logic в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

Our software went from over **1000 lines to just over 300**. This was not feature loss. Most of the deleted complexity had been compensating for the imprecise steering documented in Section 1.3, and once that was fixed in hardware, the software fighting it became dead weight.

What remains, in roughly 300 lines, still performs PID lane-centering, RGB color detection, symmetric curve execution, two-stage wall protection, corner and lap counting, time-based parking, sensor validation, and graceful failure handling. Simple does not mean it does little. It means every line earns its place.

The program is a single loop. Each 10 ms cycle reads the color sensor, reads both ultrasonic sensors, then passes the result through a five-layer priority chain. The first layer whose condition is met takes control; every layer below is skipped that cycle.

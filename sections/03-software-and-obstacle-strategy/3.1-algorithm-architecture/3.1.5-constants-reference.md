# ᯓ★ 3.1.5 Constants Reference ᯓ★

This subsection is the complete reference of every tunable value in the program, gathered in one place. Its purpose is reproducibility: another team should be able to read these tables, match them against our source, and understand exactly what every number in the code controls. Each value here is defined once at the top of the program, which is what makes the robot's behavior adjustable without hunting through the logic.

---

## ❀ Drive Speeds ────୨ৎ────────୨ৎ────

On this robot, **negative values move it forward.** This is a consequence of how the drive motor is mounted, and it is consistent across every speed constant.

| Constant | Value | Used when |
| :--- | :---: | :--- |
| `STRAIGHT_SPEED` | −62 | Normal PID centering on a straight |
| `CURVE_SPEED` | −54 | Defined for curve driving |
| `GENTLE_WALL_SPEED` | −46 | Gentle wall correction, and the single-sensor fallback |
| `SEVERE_WALL_SPEED` | −40 | Severe wall correction (closest to a wall = slowest) |
| `POST_CURVE_SPEED` | −38 | Defined for the post-curve recovery phase |

The pattern is deliberate: the closer the robot is to a wall, the slower it drives. Full speed on a clear straight, slowest during a severe wall correction. Slowing down buys the steering motor more time to respond when precision matters most.

## ❀ Steering ────୨ৎ────────୨ৎ────

| Constant | Value | Meaning |
| :--- | :---: | :--- |
| `STEERING_SPEED` | 23% | The steering motor's speed, intentionally limited |
| `POST_CURVE_STEERING_SPEED` | 40% | A faster steering speed for the recovery phase |
| `STEER_DIRECTION` | −1 | Inverts the steering sign to match the physical mounting |
| `MAX_PID_ANGLE` | 24° | Ceiling on any PID correction |
| `CURVE_ANGLE` | 55° | Defined turn angle for curve driving |
| `GENTLE_WALL_ANGLE` | 18° | Steering angle for gentle wall correction |
| `SEVERE_WALL_ANGLE` | 32° | Steering angle for severe wall correction |
| `POST_CURVE_ANGLE` | 16° | Defined counter-steer angle for recovery |

`STEERING_SPEED` is deliberately capped at 23%. A steering motor that slams to its target angle at full speed overshoots and oscillates; limiting its speed makes it settle cleanly on the commanded angle. `STEER_DIRECTION` exists because the steering motor's positive direction, as mounted, is the opposite of the sign our code produces, so this constant flips it in one place rather than scattering minus signs through the logic.

## ❀ Distances and Thresholds ────୨ৎ────────୨ৎ────

| Constant | Value | Meaning |
| :--- | :---: | :--- |
| `WALL_GENTLE_DISTANCE` | 21.5 cm | Below this, gentle wall protection triggers |
| `WALL_SEVERE_DISTANCE` | 19.0 cm | Below this, severe wall protection triggers |
| `INTEGRAL_LIMIT` | 70.0 | The clamp that prevents PID integral windup |

The two wall distances are only 2.5 cm apart. That tight gap is intentional: it gives the robot a short, graded warning zone. It corrects gently as it enters 21.5 cm, and only escalates to the sharp response if it keeps closing to 19.0 cm, rather than jumping straight to a hard correction.

| Constant | Value | Term |
| :--- | :---: | :--- |
| `KP` | 0.28 | Proportional |
| `KI` | 0.001 | Integral |

These are developed in full in 3.1.4. They are gentle by design, a result of the mechanical steering fixes in Section 1.3.

## ❀ Timing and Counting ────୨ৎ────────୨ৎ────

| Constant | Value | Meaning |
| :--- | :---: | :--- |
| `LOOP_DELAY` | 0.01 s | Time between loop cycles (≈100 cycles/second) |
| `POST_CURVE_TIME` | 0.16 s | Defined duration of the post-curve recovery window |
| `CURVES_PER_LAP` | 4 | Corners in one lap |
| `TOTAL_LAPS` | 3 | Laps in a full run |
| `TOTAL_CURVES` | 12 | Total corners (4 × 3) |
| `COLOR_CONFIRM_READINGS` | 1 | Consecutive matching readings needed to accept a color |

`TOTAL_CURVES` is computed from `CURVES_PER_LAP × TOTAL_LAPS` rather than typed as 12 directly. This is a small but deliberate choice: if the track format ever changed, we would update one number, not hunt for every place the count appears.

## ❀ Row Detection ────୨ৎ────────୨ৎ────

| Constant | Value | Meaning |
| :--- | :---: | :--- |
| `ROW_CHECK_SAMPLES` | 5 | Readings averaged to detect the starting row |
| `ROW_ALIGN_TIMEOUT` | 1.0 s | Maximum time the final alignment can run |
| `ROW_ALIGN_SPEED` | = gentle wall speed | Speed used for the final alignment nudge |
| `ROW_ALIGN_ANGLE` | = gentle wall angle | Angle used for the final alignment nudge |

The alignment reuses the gentle wall speed and angle rather than defining new values, so the final parking nudge never moves more aggressively than the robot's normal wall correction. The timeout guarantees this step cannot hang or drift the robot into extra travel. Both are covered in Section 3.3.

## ❀ A Note on the Defined-but-Unused Constants ────୨ৎ────────୨ৎ────

A few constants, `CURVE_SPEED`, `CURVE_ANGLE`, `POST_CURVE_SPEED`, `POST_CURVE_ANGLE`, `POST_CURVE_TIME`, are defined and available in the program for curve-driving and post-curve recovery routines. We document them here for completeness, since they are part of the source, and because they represent capability the program carries for the corner-driving approach described in our code's structure.

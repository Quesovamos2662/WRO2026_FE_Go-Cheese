## вқҖ Build Architecture Overview в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

Cheese v3 is built around a layered robot architecture. The lower layer contains the movement systems: the rear-wheel drive, the Ackermann steering mechanism, the wheels, and the main chassis. The middle layer holds the EV3 Brick, which acts as the main controller and power source for the EV3 motors and sensors. The upper layer supports the vision system, Arduino Nano, wiring, external lighting battery, and lamps.

This architecture was chosen because each subsystem has a different job. The drive system needs strength and traction, so it is placed low and toward the rear. The steering system needs fast and free movement, so it is placed at the front with enough clearance. The sensors need stable and unobstructed views, so their placement depends on what they are reading: wall distance, floor colors, or obstacle colors.

| Architecture Layer | Main Components | Purpose |
| :--- | :--- | :--- |
| **Lower mechanical layer** | Chassis, rear wheels, front wheels, Ackermann linkage, drive axle | Provides structure, movement, steering, and ground contact. |
| **Control layer** | EV3 Brick, Large Motor, Medium Motor, EV3-connected sensors | Controls motion, steering, wall correction, color detection, and main logic. |
| **Upper vision layer** | HuskyLens, Arduino Nano, upper lamp, wiring, 9V battery | Supports obstacle recognition and separates vision hardware from the drivetrain. |
| **Lighting layer** | Upper helping lamp and lower helping lamp | Improves camera visibility and floor color detection. |

The most important rule when rebuilding Cheese is that every subsystem must stay stable and must not interfere with another subsystem. A cable touching the steering linkage, a loose camera mount, or a tilted ultrasonic sensor can change how the robot behaves even if the code is correct.

## вқҖ Recommended Build Order в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

Cheese should be built from the bottom up. This order makes the robot easier to test because the mechanical systems can be checked before the sensors, lights, and wiring are added.

| Stage | Build Step | Reason |
| :---: | :--- | :--- |
| **1** | Build the lower chassis | Creates the rigid foundation for all subsystems. |
| **2** | Install the rear drive system | Allows propulsion to be tested before the robot becomes heavier. |
| **3** | Install the Ackermann steering system | Allows steering centering and linkage freedom to be checked early. |
| **4** | Mount the EV3 Brick | Establishes the main control and power location. |
| **5** | Build the upper support tower | Creates the structure for the vision and lighting systems. |
| **6** | Install ultrasonic sensors | Adds wall-distance sensing for navigation. |
| **7** | Install the color sensor | Adds floor color detection for curve timing. |
| **8** | Install the HuskyLens and Arduino Nano | Adds obstacle recognition support. |
| **9** | Install the dual-light system | Improves camera and floor sensor reliability. |
| **10** | Wire and secure all components | Prevents cable interference and disconnections. |
| **11** | Calibrate and test in stages | Confirms each subsystem works before full runs. |

Do not start with a full competition run immediately after building. The robot should first pass mechanical checks, then motor checks, then sensor checks, then short movement tests.

## вқҖ 1. Base Chassis Construction в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

Start with the lower chassis because every other subsystem depends on this foundation. The chassis must hold the EV3 Brick, motors, wheels, sensors, upper tower, lighting system, and wiring without bending or twisting during movement.

The base should be built using Technic liftarms, beams, pins, axles, and connectors. The structure should be symmetrical from left to right because uneven alignment can make the robot drift even when the code is correct.

### Chassis Requirements

| Chassis Area | Requirement | Why It Matters |
| :--- | :--- | :--- |
| **Main frame** | Must be rigid and straight. | Prevents twisting that can affect steering and sensor alignment. |
| **Left and right sides** | Must be symmetrical. | Prevents the robot from naturally pulling to one side. |
| **Front section** | Must leave space for the Ackermann linkage. | Steering needs free movement without friction. |
| **Rear section** | Must support the Large Motor and rear wheels firmly. | Drive force must transfer cleanly to the wheels. |
| **Upper mounting points** | Must hold the tower securely. | The camera and lights should not shake during movement. |

### Chassis Build Checks

Before adding motors and sensors, check the chassis by hand:

- The frame should not twist easily.
- Pins should not be forced into place.
- The left and right sides should be aligned.
- There should be enough space for wheels, axles, and wires.
- No beam should block the future steering movement.
- The lower frame should sit flat and stable.

If the chassis is not straight at this stage, the robot will be harder to tune later because the software will be correcting a mechanical problem.

## вқҖ 2. Rear Drive System в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

The rear drive system is responsible for propulsion. Cheese uses the **EV3 Large Motor** because moving the full robot requires more torque than steering. The Large Motor is installed in the rear drive area and connected to the rear wheels through a direct 1:1 drive setup.

The drive system should be placed low in the chassis because this keeps the robot more stable. A lower drive structure helps reduce unnecessary shaking and keeps the rear wheels firmly in contact with the mat.

| Component | Placement | Function |
| :--- | :--- | :--- |
| **EV3 Large Motor** | Rear drive section | Provides propulsion torque. |
| **Rear axle / drive connection** | Between motor and wheels | Transfers rotation from motor to wheels. |
| **Rear wheels** | Back of robot | Move the robot forward. |
| **Rear tires** | On rear wheels | Provide traction on the mat. |

### Drive System Requirements

The rear wheels must spin freely without rubbing against liftarms, pins, cables, or the chassis. If the wheels rub, the motor loses speed and the robot may drift or slow down unexpectedly.

Both rear wheels should be aligned with each other. If one wheel is angled or blocked, Cheese may not drive straight even if the steering is centered.

### Drive System Checks

| Check | Correct Result |
| :--- | :--- |
| **Wheel rotation** | Rear wheels spin freely by hand. |
| **Motor stability** | Large Motor does not move or shake in its mount. |
| **Axle alignment** | Rear axle stays straight and does not bend. |
| **Tire clearance** | Tires do not touch wires or chassis pieces. |
| **Drive direction** | When tested with code, the robot moves forward as expected. |

The drivetrain should be tested before adding the upper tower. This makes it easier to find friction or alignment problems while the robot is still simple.

## вқҖ 3. Ackermann Steering System в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

The steering system uses an **EV3 Medium Motor** connected to an Ackermann-style linkage. This system controls the front wheels and allows Cheese to make smoother turns than a fixed-wheel design.

The Medium Motor is used for steering because it is lighter and faster than the Large Motor. Steering does not need to move the full robot mass; it needs quick and repeated angle changes. This makes the Medium Motor better for PID corrections, wall protection, and curve entry.

</p>

<p align="center">
  <em>Front view of Cheese v3 showing the front steering area, sensors, and lighting layout.</em>

### Steering Architecture

| Part | Purpose |
| :--- | :--- |
| **EV3 Medium Motor** | Rotates the steering mechanism. |
| **Ackermann linkage** | Transfers motor movement to both front wheels. |
| **Front wheels** | Control the robotвҖҷs direction. |
| **Steering connectors and axles** | Keep the front wheels linked and aligned. |

The steering linkage must move freely. If it is too tight, the Medium Motor may not reach the commanded angle in time. This can cause Cheese to enter curves too straight or crash into a wall.

### Steering Centering

Before software testing, the front wheels must be mechanically centered. A wrong center position makes the robot drift even if the PID values are correct.

To check steering center:

1. Move the front wheels to the straightest possible position.
2. Make sure both front wheels point forward evenly.
3. Confirm the Medium Motor is not pushing the linkage to one side.
4. Run a small steering test.
5. Confirm the steering returns to center after turning left and right.

### Steering Checks

| Check | Correct Result |
| :--- | :--- |
| **Left steering** | Wheels turn left smoothly. |
| **Right steering** | Wheels turn right smoothly. |
| **Return to center** | Wheels return to a straight position. |
| **Linkage clearance** | No liftarm, pin, or wire blocks the movement. |
| **Motor mount** | Medium Motor stays fixed while steering. |
| **Symmetry** | Left and right steering angles are similar. |

Steering should be tested before connecting all sensors because cable interference is one of the easiest ways to make the steering slower or inconsistent.

## вқҖ 4. EV3 Brick Placement в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

The **EV3 Brick** is the main controller of Cheese. It also powers the EV3 motors and EV3-connected sensors. Because of its weight, the EV3 should be placed in a stable central area of the chassis instead of being mounted too high or too far to one side.

A balanced EV3 placement helps Cheese maintain stability during turns and wall corrections. If the EV3 is poorly placed, the robot may tilt, vibrate, or place too much load on one side.

| EV3 Placement Requirement | Reason |
| :--- | :--- |
| **Central position** | Helps balance the robotвҖҷs weight. |
| **Firm mounting** | Prevents the EV3 from shifting during movement. |
| **Accessible ports** | Makes wiring, debugging, and repairs easier. |
| **Clear cable routes** | Reduces the chance of cables reaching wheels or steering. |
| **Low enough placement** | Keeps the center of mass more stable. |

The EV3 should not block the steering linkage, rear wheels, sensors, or upper tower supports. It should also be easy to access for turning on the robot, changing batteries, connecting cables, and uploading code.

## вқҖ 6. Ultrasonic Sensor Installation в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

Cheese uses two ultrasonic sensors for wall-distance detection. These sensors support the PID centering system and wall protection logic. The left and right ultrasonic sensors should be mounted on the sides of the robot with a clear view of the walls.

| Sensor | Port | Placement | Purpose |
| :--- | :---: | :--- | :--- |
| **Left ultrasonic sensor** | S3 | Left side/front-side area | Measures distance from the left wall. |
| **Right ultrasonic sensor** | S2 | Right side/front-side area | Measures distance from the right wall. |

### Ultrasonic Placement Rules

The ultrasonic sensors must be stable and unobstructed. If the sensor face is blocked by a wire or LEGO piece, the readings can become false. If the sensor angle changes during movement, the robot may think it is closer or farther from the wall than it actually is.

| Rule | Reason |
| :--- | :--- |
| **Keep sensor faces clear** | Prevents blocked or false readings. |
| **Mount sensors firmly** | Prevents changing readings during movement. |
| **Avoid extreme angles** | The sound wave may bounce away from the wall. |
| **Keep left and right sensors comparable** | Helps PID interpret wall distance correctly. |
| **Keep wires behind the sensor face** | Prevents cables from interfering with readings. |

### Ultrasonic Checks

Before full runs, place Cheese between two walls and observe both sensor readings. The values should change smoothly when the robot is moved left or right. If readings jump suddenly, check the sensor angle, cable position, and nearby LEGO pieces.

## вқҖ 7. Color Sensor Installation в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

The color sensor is mounted in the lower front area and reads the mat surface. In Cheese v3, this sensor supports curve timing by detecting floor colors. Because it reads the floor, its height and lighting are very important.

| Requirement | Reason |
| :--- | :--- |
| **Low placement** | Keeps the sensor close enough to read the mat accurately. |
| **Stable height** | Prevents color readings from changing while driving. |
| **Clear floor view** | Prevents shadows or LEGO parts from blocking readings. |
| **Lower lamp support** | Improves color consistency under bad lighting. |
| **No cable obstruction** | Prevents shadows and physical interference. |

The color sensor should not be mounted too high. If it is too high, it may read colors inconsistently. If it is too low, it may touch the mat or get blocked during movement. The best position is close enough to read the floor clearly while still leaving safe clearance.

### Color Sensor Checks

| Check | Correct Result |
| :--- | :--- |
| **Floor visibility** | Sensor has a clear view of the mat. |
| **Height stability** | Sensor height does not change while moving. |
| **Color detection** | Blue/orange or required floor colors are detected consistently. |
| **Lower light alignment** | Floor near the sensor is illuminated. |
| **Cable clearance** | Cable does not drag or block the sensor. |

If the sensor misses colors, check the physical setup first before changing the code. Many color problems come from height, shadows, or poor lighting.

## вқҖ 8. HuskyLens and Arduino Nano Installation в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

The HuskyLens camera supports obstacle recognition. The Arduino Nano acts as a communication bridge between the HuskyLens and the EV3-based robot architecture.

The HuskyLens should be mounted on the upper front area of Cheese with a clear view of the obstacle field. The Arduino Nano should be mounted close enough to the camera to keep I2C wiring short and organized, but it should not block the camera or interfere with the tower structure.

### Vision System Architecture

| Component | Placement | Function |
| :--- | :--- | :--- |

| **Arduino Nano** | Upper/side wiring area | Works as a bridge for HuskyLens communication. |

### HuskyLens Placement Rules

The camera should be high enough to see obstacles before the robot reaches them, but not so high or tilted that it misses the obstacle area. The lens must stay clear of wires, lamps, liftarms, and the 9V battery.

| **Correct angle** | The camera must face the obstacle area, not the floor or ceiling. |
| **Upper light support** | Better lighting improves red and green recognition. |
| **Short organized wiring** | Reduces disconnection risk and cable movement. |

## вқҖ 9. Dual-Light Support System в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

Cheese uses a dual-light support system because the HuskyLens and color sensor need light in different areas. The upper lamp supports the camera, while the lower lamp supports the floor color sensor.

<p align="center">
  <img src="../../v-photos/v3/dual_light_system_v3.jpeg" alt="Cheese v3 dual-light support system" width="75%">
</p>

<p align="center">

| Lamp | Placement | Sensor Supported | Purpose |
| :--- | :--- | :--- | :--- |
| **Upper helping lamp** | Upper front area | HuskyLens camera | Improves obstacle visibility and color recognition. |
| **Lower helping lamp** | Lower front area, near the floor | Color sensor | Improves floor illumination and color reading consistency. |
| **9V battery** | Upper support structure | Both lamps | Powers the lights separately from the EV3. |

The lamps should be powered by the external **9V battery**, not by the EV3. This keeps the lighting system separate from the EV3 power system, which already powers the motors, sensors, and main control logic.

### Lighting Checks

| Check | Correct Result |
| :--- | :--- |
| **Upper lamp angle** | Illuminates the obstacle area for HuskyLens. |
| **Lower lamp angle** | Illuminates the mat area near the color sensor. |
| **Battery stability** | 9V battery does not move during driving. |
| **Wire safety** | Light wires do not touch wheels or steering. |
| **Sensor improvement** | Camera and color readings become more stable with lights on. |

If only one lamp is used, one sensor may improve while the other remains unstable. That is why the final architecture uses two lights instead of one.

## вқҖ 10. Wiring and Port Map в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

The wiring must match the robotвҖҷs code. If a sensor or motor is plugged into the wrong port, the program may run but the robot will behave incorrectly.

| Device | Port / Connection | Purpose |
| :--- | :--- | :--- |
| **EV3 Large Motor** | Output B | Rear propulsion. |
| **EV3 Medium Motor** | Output A | Ackermann steering control. |
| **Left Ultrasonic Sensor** | Input S3 | Left wall detection. |
| **Right Ultrasonic Sensor** | Input S2 | Right wall detection. |
| **Color Sensor** | Input S4 | Floor color detection and curve timing. |
| **Arduino Nano** | EV3 USB / communication setup | Vision bridge. |
| **HuskyLens** | Arduino Nano I2C pins | Red and green obstacle recognition. |
| **Upper helping lamp** | External 9V battery circuit | Camera lighting support. |
| **Lower helping lamp** | External 9V battery circuit | Floor color sensor lighting support. |

> **Important:** The physical wiring must match the code. If any port changes, the code must be updated before testing.

Related file:  
[`2.2-wiring-diagram.md`](../02-power-and-sensor-architecture/2.2-wiring-diagram.md)

### Wiring Order

Wire Cheese in this order:

1. Connect the EV3 Large Motor to Output B.
2. Connect the EV3 Medium Motor to Output A.
3. Connect the left ultrasonic sensor to S3.
4. Connect the right ultrasonic sensor to S2.
5. Connect the color sensor to S4.
6. Connect the Arduino Nano to the EV3 communication setup.
7. Connect the HuskyLens to the Arduino Nano using I2C wiring.
8. Connect the upper and lower lamps to the external 9V lighting circuit.
9. Secure all wires before powering the robot.

This order helps separate movement wiring, sensor wiring, vision wiring, and lighting wiring.

## вқҖ 11. Cable Management в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

Cable management is part of the build, not just decoration. Loose wires can block sensors, pull on the steering linkage, touch the wheels, or disconnect during testing.

<p align="center">
  <img src="../../v-photos/v3/back_v3.jpg" alt="Cheese v3 back view" width="75%">
</p>

<p align="center">

### Cable Management Rules

| Rule | Why It Matters |
| :--- | :--- |
| **Keep cables away from rear wheels** | Prevents friction and unexpected stops. |
| **Keep cables away from Ackermann linkage** | Prevents steering resistance or delayed steering. |
| **Do not block ultrasonic sensors** | Keeps wall readings reliable. |
| **Do not block the HuskyLens lens** | Keeps obstacle recognition clear. |
| **Do not cover the color sensor** | Keeps floor detection consistent. |
| **Secure wires on the upper tower** | Reduces disconnections and cable movement. |
| **Leave controlled slack where needed** | Prevents cables from pulling during turns. |

The steering area should be checked after cable routing. A wire can look harmless while the robot is stopped, but pull on the steering mechanism during turns.

## вқҖ 12. Pre-Run Calibration в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

Before running Cheese on the full mat, calibrate the mechanical, sensor, lighting, and software systems. Calibration should happen after every major rebuild or sensor repositioning.

| Calibration Area | What to Check | Correct Result |
| :--- | :--- | :--- |
| **Steering center** | Front wheels point straight when centered. | Robot does not drift strongly on a short straight test. |
| **Large Motor direction** | Robot moves forward when drive motor runs. | Forward command produces forward movement. |
| **Medium Motor direction** | Steering turns in the expected direction. | Left/right steering matches the code. |
| **Ultrasonic readings** | Left and right sensors return stable distances. | Readings change smoothly when robot position changes. |
| **Color sensor readings** | Track colors are detected under the lower lamp. | Floor colors are recognized consistently. |
| **HuskyLens recognition** | Red and green obstacles are detected correctly. | Camera identifies the correct color/object. |
| **Upper lamp angle** | Obstacle colors are visible to the camera. | HuskyLens view improves with the lamp on. |
| **Lower lamp angle** | Floor is illuminated near the color sensor. | Color sensor readings become more stable. |
| **Cable clearance** | No wires touch wheels or steering. | Movement is not blocked by cables. |

Calibration is not only a software step. Most navigation problems should be checked mechanically first because bad alignment can make correct code behave incorrectly.

## вқҖ 13. Testing Workflow в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

Test Cheese in stages. This makes debugging easier because each problem can be connected to one subsystem instead of the entire robot.

| Test Stage | What to Test | What Should Happen |
| :---: | :--- | :--- |
| **1** | EV3 startup | Program starts correctly and waits for start input. |
| **2** | Large Motor only | Rear wheels move in the correct direction. |
| **3** | Medium Motor only | Front wheels steer left, right, and return to center. |
| **4** | Ackermann linkage | Steering moves smoothly without jamming. |
| **5** | Ultrasonic sensors | Left and right distances are readable and stable. |
| **6** | Color sensor | Floor colors are detected consistently. |
| **7** | HuskyLens | Red and green obstacles are recognized. |
| **8** | Dual-light system | Camera and floor readings improve with lights on. |
| **9** | Short straight run | Robot moves forward without strong drifting. |
| **10** | Wall-following test | PID keeps the robot between the walls. |
| **11** | Curve test | Robot enters and exits curves without hitting walls. |
| **12** | Post-curve recovery test | Robot stabilizes after leaving a turn. |
| **13** | Full open round test | Robot completes the required laps. |
| **14** | Parking test | Robot stops in the intended area. |
| **15** | Obstacle test | Robot detects obstacles and reacts correctly. |

Do not tune everything at the same time. If Cheese crashes after a curve, test curve entry, steering response, and post-curve recovery before changing unrelated sensor values.

## вқҖ 14. Common Problems and Fixes в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

| Problem | Possible Cause | Suggested Fix |
| :--- | :--- | :--- |
| **Robot pulls to one side** | Steering center is off, chassis is uneven, or wheels are misaligned. | Recenter steering and check wheel alignment. |
| **Steering gets stuck** | Linkage friction, tight pins, or cable interference. | Check linkage movement and reroute cables. |
| **Robot oscillates between walls** | PID values are too aggressive or ultrasonic readings are unstable. | Tune PID and check ultrasonic mounts. |
| **Robot crashes after curves** | Poor curve exit angle or weak recovery logic. | Tune curve behavior and post-curve recovery. |
| **Curves are too wide** | Steering angle or curve timing is too weak. | Increase curve steering carefully and retest. |
| **Color sensor misses colors** | Sensor is too high, floor is too dark, or lower lamp is misaligned. | Adjust sensor height and lower lamp angle. |
| **HuskyLens misses obstacles** | Camera angle, lighting, or training is incorrect. | Adjust camera, upper lamp, and training settings. |
| **Cables interfere with movement** | Wires are loose near wheels or steering. | Secure cables through the upper frame. |
| **Pins break or loosen** | Too much pressure on the connection. | Reinforce the joint and avoid forcing pins. |
| **Parking fails** | Stop logic, final reference, or approach speed is not tuned. | Test parking separately after lap navigation. |

A common mistake is changing the code before checking the build. For Cheese, mechanical alignment, sensor placement, lighting, and cable clearance must be checked before assuming the program is wrong.

## вқҖ Final Build Note в”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖаӯЁа§Һв”Җв”Җв”Җв”Җ

Cheese v3 should not be rebuilt only by copying its shape. To reproduce it correctly, the builder must also reproduce the purpose of each subsystem: stable rear-wheel propulsion, responsive Ackermann steering, reliable wall sensing, consistent floor color detection, supported obstacle recognition, separated lighting power, and safe cable management.

The physical robot and the software depend on each other. If the motors, sensors, lights, or cables are placed differently, the robot may need different calibration or code values. For this reason, rebuilding Cheese requires both mechanical accuracy and testing discipline.

<p align="center">
  <strong>A reproducible robot is not only one that can be rebuilt, but one that can be rebuilt, tested, understood, debugged, and improved.</strong>
